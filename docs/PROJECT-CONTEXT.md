# exist — 프로젝트 컨텍스트

> 이 문서는 새로 합류하는 사람/AI가 exist의 **비전·구조·현황**을 한 번에 파악하도록 정리한 것이다.
> 코드 세부는 각 디렉터리, 테스트 현황은 [`QA-SCENARIOS.md`](./QA-SCENARIOS.md) 참고.
> (배포 인프라의 민감 정보 — 서버 주소·계정·키 — 는 보안상 이 public 저장소에 두지 않는다.)

---

## 1. 한 줄 정의

**exist는 분산 근무를 디지털화(DX)하지 않는다. AI로 재정의(AX)한다.** (7/6 확정: "재택근무" 아닌 "분산 근무" — 재택은 ESG 파트 하위 개념으로만)

Teams·Slack·Zoom이 오프라인 업무를 디지털로 "옮긴" 도구(DX)라면, exist는 **AI가 팀 운영의 주체가 되는** 플랫폼(AX)이다. 사람은 판단·창작에 집중하고, 운영·조율·맥락관리는 AI가 맡는다.

## 2. 비전 — AX 플랫폼의 3기둥

세 기둥 모두 **"AI가 보조가 아니라 운영 주체"** 라는 한 줄로 꿰인다.

| 기둥 | 사람이 잃은 것 | DX(Teams) | AX(exist) — AI가 주체 |
|---|---|---|---|
| **A. 지속가능성** | 팀이 조용히 번아웃 | 안 보임 | AI가 번아웃·과부하·고립을 **선제 감지·개입** |
| **B. 연속성** | 비동기로 일하면 흐름 단절 | 로그만 쌓임 | AI가 "어젯밤 팀 스토리"로 **흐름을 이어줌** |
| **C. 맥락** | 합류·복귀하면 맥락 0 | 사람이 찾아 헤맴 | AI가 맥락 **자동 보존·온보딩** |

**증명 전략:** 비전은 3기둥을 다 그리되, 실물 MVP는 **A(지속가능성)** 를 깊게 판다 — 이미 AI 인사이트(번아웃/지연 예측)+ESG 자산이 있어, "감지"에서 "개입"까지 확장하면 완성된다. (버즈워드를 피하려면 "AX"를 말로만 하지 않고 한두 기둥을 실물로 강하게 증명해야 한다.)

## 3. 기술 스택 / 구조

- **프론트엔드:** React + Vite + TypeScript (`client/`)
- **백엔드:** Express 5 + Socket.IO + better-sqlite3 (`server/`)
- **실시간 통화:** mediasoup SFU 직접 구현 (`server/src/sfu.ts`)
- **공동 편집:** Yjs CRDT — 캔버스(Excalidraw)·코드(CodeMirror)·문서(Tiptap)·시트·슬라이드 5종 (`server/src/ydoc.ts`)
- **AI:** OpenAI(gpt-4o-mini) — 팀 인사이트/브리핑 (`server/src/insights.ts`, `agent.ts`)
- **코드 실행:** 격리된 별도 컨테이너(`runner/`) — internal 네트워크·read-only·non-root·cap_drop로 RCE 방지
- **배포:** Docker (Mac mini Colima) + nginx 리버스 프록시. compose는 `deploy/`

### 주요 디렉터리
```
client/src/components/   UI 컴포넌트
server/src/              API 라우터·소켓·SFU·Yjs·AI
server/src/app.ts        express app 생성(테스트용 분리)
server/src/__tests__/    vitest 단위·통합 테스트
runner/                  격리 코드 실행 서비스
deploy/                  배포 compose (compose.macmini.yml = 라이브)
docs/                    이 문서, QA 시나리오
```

## 6. 방향성 — "협력 강화" 프레이밍 (2026-07-17 확정)

### 예시 기업: HK이노엔 (공개 정보 기반 구조 예시)
공식 홈페이지(inno-n.com) 확인 팩트:
- **본사·오송공장 = 충북 청주 오송** (본사가 서울이 아님!)
- **R&D = 판교** (HK이노엔 스퀘어), **대소공장 = 충북 음성**, 이천공장 = 경기
- 사무소 = 서울 3곳(노원·상암·서초) 포함 **전국 13개 지역**

→ 본사가 법적으로 충북에 있어도 지식노동(R&D·기획·영업)은 수도권에 배치된다.
"기업 유치하면 되잖아?"라는 반론을 실존 사례로 깨는 구조.
⚠️ 서류에 "고객/협의 중"처럼 보이게 쓰지 말 것 — "충북 기업의 전형적 구조 예시"로만.

### 서사 (문제 → 답)
1. **문제(현재 팩트)**: 오송 본사·공장 ↔ 판교 R&D ↔ 서울 사무소로 찢어진 사업장 간 **협력 단절**
   - 현장–R&D 맥락 단절, 정보는 수도권 쪽에 고임, 충북 사업장은 "결정을 통보받는 2등 사업장"
2. **exist의 답**: AI 운영 레이어가 사업장 간 **협력을 강화** → 충북 거점이 본사·R&D와 **대등**해진다
3. 뿌리내림·정주 효과는 **장기 비전 한 줄로만** (멘토 피드백: 미래 계획 줄여라)

### 표현 규칙
- ❌ "유출" 금지 (판교 R&D는 충북에서 빠져나간 게 아님 — 연혁 미확인)
- ❌ 협력 강화만 말하면 Teams/Slack과 비교당함 → **AX 차별점(AI가 맥락·연속성을 들고 팀 운영을 주도)을 문제 정의 바로 옆에 배치**
- ❌ 단정적 추측 수치 금지 — 통계는 통계청·공공데이터포털 실데이터만 (청주시 산업단지 입주기업 목록 1,232건 CSV 활용 가능)
- ❌ "한국형" 금지 (근거 기능 없이 국가명 붙이면 입증 부담만 생김) → **"현장·교대근무를 포함한 분산 조직용"**으로
- ❌ "협력사 간 소통"은 v1 서사에 넣지 않음 (계정·보안 설계가 다른 문제, 현재 제품에 없음) → 로드맵 한 줄로만

### 포지셔닝 문장 (7/17 확정) — 독창성 방어의 축
> "글로벌 협업 도구(Teams·Zoom·Webex·Google Workspace)는 책상 앞 사무직을 위해 설계됐다.
> exist는 **본사 사무직과 지방 공장 현장직이 한 팀인 한국 중소·중견 제조업의 분산 조직**을 위해 설계됐고,
> 회의에 들어오지 못하는 사람에게 AI가 결정과 맥락을 배달한다."

- vs Copilot/Zoom AI 반격 논리: **"Copilot은 회의에 들어온 사무직을 돕고(물어보면 답하는 비서),
  exist는 회의에 못 들어오는 현장직까지 팀으로 잇는다(시키지 않아도 챙기는 총무)."**
- 타겟: 전담 IT팀 없고 Copilot 라이선스가 부담인 **중소·중견 제조업** (대기업은 Teams에 묶여 있음 — 정면승부 X).
  충북에 많은 기업 유형이라 지역 서사와도 일치.

### 심사 항목 자가진단 (7/17, 주관적)
| 항목 | 상태 | 핵심 |
|---|---|---|
| 적합성 | 강 | AI×협업×지역 — 취지 정면. 단 AI가 부가기능으로 보이면 무너짐 |
| 기술성 | 강 | 플랫폼 엔지니어링 상위. AI 파트가 얕음 → P1을 "맥락 라우팅 파이프라인"으로 설계 |
| 사업화 | 중 | 완성도 물증(라이브·QA98) 있음. 시장 논리·BM은 백지 → 사업계획서에서 채울 것 |
| 독창성 | 약→보강중 | Copilot 기시감이 최대 리스크 → 위 포지셔닝 문장 + P1 실물로 방어 |

→ 네 항목 모두 **P1 구현**이 공통 지렛대. P1 없이 서류만 쓰면 독창성에서 무너짐.

## 7. 해야 할 일 (우선순위순)

협력 강화 서사의 기능 우선순위. 도구(화상·공동편집·DM·인박스)는 이미 있으므로,
**전부 "기존 인박스·알림·회의 위에 AI가 흐름을 잇는다"로 좁게** 잡는다 (멘토: AI는 간단히만).

### P1. 회의 → 결정 → 전달 파이프라인 (연속성) — ✅ 구현 완료 (2026-07-17, 로컬 검증)
- 통화 방이 비면(유예 `RECAP_GRACE_MS`, 기본 30초) → 그 세션 채팅에서 AI가 결정·할 일 추출
  (gpt-4o-mini + 규칙 폴백) → `meeting_recaps` 저장 → **참석/불참 구분해 알림 라우팅**
  ("놓친 통화의 결정이 도착했어요") → @멘션·자기약속 담당자는 **할 일 자동 생성**(홈 목록에 회의 칩과 함께 표시)
- 구현: `server/src/recap.ts`(신규), sfu 종료 훅, `GET /:code/recaps`, `RecapPanel.tsx`(허브 대시보드 "AI 회의 정리" 카드, 실시간 갱신), 알림 "정리 보기" 버튼
- 검증: 단위·통합 14테스트 + 실소켓 E2E 통과. **남은 것: 배포 후 라이브 2브라우저 확인(RCP-06)**
- 시연 컷: 본사(판교 역할) 통화 종료 → 충북지사 직원 화면에 알림 토스트 + 홈 할 일에 배정 + 허브 recap 카드

### P2. 비동기 브리핑 (맥락) — ✅ 구현 완료 (2026-07-17, 로컬 검증)
- `users.last_seen_at`(마지막 접속 종료) 이후의 **놓친 recap·새 할 일·안읽은 DM·안읽은 그룹 채팅**을
  규칙으로 집계(AI가 지어낼 수 없는 구조) + AI 헤드라인 한 줄 → 홈 "AI 피드백" 카드에 표시, 항목 클릭 시 해당 회의 열림
- 구현: `GET /api/agent/catchup`, ProfileDashboard 개편
- 검증: 통합 3테스트 통과. 남은 것: 배포 후 UI 확인(CTU-04)

### P3. 조직 맥락 질의응답 (맥락) — 서류 중심, 데모 선택
